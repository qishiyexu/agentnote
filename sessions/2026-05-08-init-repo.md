## 话题
搭建一个能"沉淀人与 AI Agent 交互"的 Git 仓库，要求换机器、换 Agent 都能无缝衔接。

## 进展
1. 明确目标：不是"给某一款 Agent 配置 memory"，而是把交互产出**工程化**——以 Markdown + Git 为载体，任何 Agent 都能读写。
2. 讨论了选型：放弃 Notion / Obsidian / 数据库方案，选择"纯文本 + Git"，理由见 [0001](../decisions/0001-use-markdown-and-git.md)。
3. 设计了文档分类：sessions / topics / decisions / references 四类，职责单一，理由见 [0002](../decisions/0002-four-doc-types.md)。
4. 写了 [AGENTS.md](../AGENTS.md)：作为所有 Agent 接手仓库的"入职手册"，明确三条铁律、标准流程、提交规范、反模式。
5. 写了 [INDEX.md](../INDEX.md)：作为全局入口，所有 Agent 从这里出发。
6. 在 `templates/` 下提供 4 个模板，降低后续写作摩擦。

## 结论
- **存储介质**：Markdown + Git（[0001](../decisions/0001-use-markdown-and-git.md)）。
- **文档分类**：sessions / topics / decisions / references（[0002](../decisions/0002-four-doc-types.md)）。
- **协作流程**：接手时读 AGENTS.md → INDEX.md → 最近 session；离开前 commit + push + 更新 INDEX.md。
- **Agent 切换约定**：任何 Agent 都视为"新人"，靠文档而非记忆衔接。
